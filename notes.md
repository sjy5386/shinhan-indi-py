# INDI TR 사양 메모 — 포트폴리오 분석용

전체 사양은 `Special_Interface.md` 참조. 이 파일은 실제 사용하는 TR의 핵심 필드만 발췌 + **실측에서 발견한 사양서와의 불일치/워닝**.

## 환경

| 항목 | 값 |
|-----|----|
| 32bit ProgID | `GIEXPERTCONTROL.GiExpertControlCtrl.1` (이미 등록) |
| 32bit OCX | `C:\SHINHAN-i\indi\giexpertcontrol.ocx` |
| 64bit ProgID | `GIEXPERTCONTROL6.GIExpertControl6Ctrl.1` (regsvr32 등록 필요) |
| 64bit OCX | `C:\SHINHAN-i\indi\giexpertcontrol64.ocx` |
| Starter | `C:\SHINHAN-i\indi\giexpertstarter.exe` |

## 로그인 (Comm_Indi 메서드)

- `StartIndi(ID, PW, CERT_PW, STARTER_PATH)` → bool
- `CloseIndi()` → bool

비동기로 진행되므로 호출 후 `GetCommState()` 폴링 또는 `ReceiveSysMsg` 이벤트 대기.

## TR 호출 패턴

```
SetQueryName("TR_CODE")
SetSingleData(idx, value)         # 입력 Single
SetMultiData(row, col, value)     # 입력 Multi (필요 시)
rqid = RequestData()              # 호출
# → ReceiveData(rqid) 이벤트로 결과 수신
GetSingleData(idx)                # 출력 Single
GetMultiRowCount()                # 출력 Multi 행 수
GetMultiData(row, col)            # 출력 Multi 셀
```

### v0.6 실측 결과 (2026-05-16)

**결론: SAAA612QB는 v0.6 후보로 폐기.** 시세조회 + 인증서 모드 모두 0행.

| 모드 | 시도 | 결과 |
|------|------|------|
| 시세조회 | 8계좌 전수 호출 | 모두 `rows=0` |
| 시세조회 | z622.foreign_stock_eval > 0 계좌 (해외주식 실보유) | `rows=0` |
| 시세조회 | 처리일자 today vs T-1 영업일 (토→금) | 모두 `rows=0` |
| 시세조회 | 계좌상품코드_in "01" vs "" | 모두 `rows=0` |
| **인증서** | 8계좌 전수 호출 | **모두 `rows=0`** |

TR 응답 자체는 정상 — `rqid` 양수, `ReceiveData` 수신, `elapsed=15-170ms`. 에러 메시지 없음. **빈 응답.** 시세조회 모드 권한 차등 가설 기각.

**진단:** SAAA612QB가 이 사용자/계좌 환경에서 영구 0행. "특정일잔고확인서"가 진짜 영업점/감리 전용 서식이거나, 사양서 INPUT 불완전, 또는 종합계좌(부모) 전용일 가능성. 더 이상 변수 통제로 결판 불가.

**사양서 supply-side 발굴 종료:** "해외주식 잔고" / "외화증권" / "해외 보유" 키워드 매칭 X. SABA(국내 위탁) / SABZ(자산 합계) 시리즈 외 해외주식 종목별 단발 TR 없음. SOL HTS 2700("해외주식 잔고")은 신한 신형 API — INDI(구형 OCX) 미지원.

**유일한 후보: AG 실시간 RT** (변경이력 L115에만 등재, 본 사양 누락) → v0.7로 분리.

코드는 보존 (`fetch_saaa612qb_holdings` + 통합) — 비활성으로 남기되 사용성 X. 다음 정리 라운드에서 롤백 결정 가능.
